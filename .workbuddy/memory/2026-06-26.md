## 第二轮修复：音乐盒拾取导致误切换到其他地图 (14:30)

### 问题诊断
第一轮修复中，`TryAutoDetectMap` 的「第三步」在音乐盒匹配到不同地图时会立即切换。当音乐盒被拾取后，如果场景中还有其他音乐盒（残留数据），
代码会匹配到其他地图并直接切换，导致从「地图1」误跳到「地图2」。

### 修复方案：添加地图切换冷却机制
新增变量：
```cpp
static int g_switch_candidate_index = -1;       // 候选切换目标
static int g_switch_confirm_frames = 0;         // 连续匹配帧数
static const int SWITCH_CONFIRM_REQUIRED = 30;  // 需连续30帧才确认
```

**核心逻辑**：当音乐盒匹配到与当前地图**不同**的地图时，不立即切换；
需要该候选地图被**连续匹配30帧（~0.5秒）**后才真正执行切换。
这防止了单帧残留数据导致的误切换。

### 完整重写 TryAutoDetectMap 五步流程：
1. **扫描音乐盒** → 同前
2. **未找到音乐盒** → 宽限期 + 玩家坐标兜底（保持地图10秒）
3. **找到+匹配DB**：
   - 匹配当前地图 → 立即确认
   - 匹配不同地图 → 启动冷却计时器，连续30帧后才切换
4. **找到+不在DB**：
   - 情况A：玩家在已知地图 → 标记音乐盒移动
   - 情况B：玩家在其他DB地图 → 切换
   - 情况C：完全新地图 → 显示居中弹窗提示

### 按钮位置优化
- 将「★ 注册当前场景为新地图」按钮移到地图管理tab顶部，**始终可见**
- 更新居中弹窗提示文字：「切换到「地图管理」菜单，点击「设为新地图」按钮」

## 第五轮：四个 Bug 修复（15:55）

### 1. 路径绘制不能弯曲
- 根因：`g_ortho_draw` 默认 `true`，强制所有线段水平/垂直
- 修复：改为默认 `false`（自由模式），支持任意方向平滑曲线
- 额外：减小 `g_path_draw_threshold`（25→15）和 `g_smooth_strength`（8→4）

### 2. 绘制路径不可见
- 根因：`Draw_MapOverlay()` 入口处同时检查 `g_map_enabled` 和 `MjSubsystem::draw_props`
- 修复：移除 `!MjSubsystem::draw_props` 条件，路径绘制不再依赖摸金模式

### 3. 注册新地图按钮过窄
- 根因：按钮宽度使用 `GetContentRegionAvail().x`，新全局 padding 下不够宽
- 修复：用 `g_font_ui` 大字 + 最小宽度确保文字完整 + 增加高度 42*density

### 4. 拾取音乐盒仍导致地图变化
- 根因：音乐盒坐标匹配到不同地图时，即使玩家还在当前地图也启动切换冷却
- 修复：在匹配到不同地图时，先检查玩家是否仍在当前地图 — 是则直接忽略
- 额外：音乐盒消失时重置 `g_frames_since_musicbox_lost` 避免累积超时

## 第六轮：正交绘制方向修复 + 路线导航指引（16:09-16:24）

### 1. 正交模式方向锁定修复

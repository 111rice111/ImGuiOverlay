# 更新日志

## [2026-06-26] v1.2 → v1.3

### 🐛 Bug 修复

**10. 地图选择器始终可见**
- 修复：原来地图选择器只在 `!g_map_auto_detect` 时显示
- 改为：无论自动识别是否开启，手动选择器始终显示
- 用户可随时覆盖自动识别的错误结果
- 相关文件：`draw_Gui.cpp`

**11. 2楼自动识别失效修复**
- 根因1：多信号评分匹配选择 `matched_db_entry` 时，只匹配 `mapIndex` 不匹配 `floorIndex`，
  始终选中一楼（第一个）的音乐盒条目，导致楼层始终被设为 0
- 根因2：钢琴兜底切换（piano-only fallback）始终设置 `g_current_floor_index = 0`
- 根因3：玩家坐标兜底（情况B）和音乐盒丢失（第二步）的 fallback 路径同样设置 floor=0
- 修复：所有路径统一根据玩家 Z 坐标 `FLOOR_Z_THRESHOLD=190.0f` 判断楼层
- 新增：`ITEM_Z_FILTER=175.0f` 统一物品过滤的 Z 轴阈值
- 相关文件：`draw_Gui.cpp`

### ✨ 新功能

**12. 手动地图选择器（始终可见）**
- 地图管理界面新增下拉列表，显示所有已配置地图
- 自动识别错误时，用户可手动选择正确地图
- 选择后自动加载对应地图纹理
- 相关文件：`draw_Gui.cpp`

**13. 场景物体一键同步**
- 新增「[同步] 物体到当前地图」按钮
- 自动保存当前场景的钢琴+凳子坐标到该地图配置
- 不需要手动指定"这个物体属于哪个地图"
- 相关文件：`draw_Gui.cpp`

## [2026-06-26] v1.0 → v1.1

### 🐛 Bug 修复

**1. SaveConfig 配置文件覆盖保护**
- 修复：`SaveConfig()` 在 `g_all_maps` 为空时不再截断覆盖已有配置文件
- 防止启动初期地图数据未加载时丢失校准参数
- 相关文件：`draw_Gui.cpp`

**2. 音乐盒匹配改进：评分制 + 坐标交叉验证**
- 修复：原来"先到先得"匹配导致地图1被误判为地图2
- 改为评分制：收集所有容差内的候选条目，按距离+坐标验证综合评分
- 新增歧义保护：多地图匹配时拒绝切换
- 相关文件：`draw_Gui.cpp`

**3. 重连后地图状态残留**
- 修复：`g_need_reinit` 触发时未重置地图状态变量
- `g_current_map_index`、`g_last_valid_map_index` 等10+变量全部残留
- 修复：重连时清除纹理缓存 + 重置全部地图索引
- 相关文件：`draw_Gui.cpp`

**4. 纹理加载不检测地图变化**
- 修复：`Draw_MapOverlay` 原来只跟踪楼层变化，地图切换时不加载纹理
- 改为同时跟踪地图+楼层索引变化
- 相关文件：`draw_Gui.cpp`

### ✨ 新功能

**5. 配置文件滚动备份**
- 每次 SaveConfig 自动备份最近3个版本
- 路径：`/sdcard/maps/calib_backup/overlay_config.{bak,bak1,bak2}`
- 相关文件：`draw_Gui.cpp`

**6. GL 上下文丢失检测**
- 每60帧用 `glIsTexture` 验证纹理ID有效性
- 检测到丢失后自动清空缓存并重新加载
- 相关文件：`draw_Gui.cpp`

**7. 多信号地图识别（钢琴融合）**
- 新增 `g_piano_db` 钢琴位置数据库（第二信号源）
- 音乐盒+钢琴加权评分：音乐盒(1.0) + 钢琴(0.8) + 坐标验证(+2.0)
- 音乐盒丢失时钢琴兜底切换
- 新增 `g_detected_piano_pos` 自动检测
- 弹窗新增钢琴 X/Y/Z 编辑字段
- 相关文件：`draw_Gui.cpp`

**8. 路径绘图工具增强**
- 路径持久化：JSON 存储于 `map_config.json` 的 `player_paths` 字段
- 左键点击路径高亮选中（亮黄色加粗）
- 删除方式：`删除选中路径` 按钮 + Delete 键
- 点击空白区域取消选中
- 移除网格吸附功能（精确点击定位）
- 文件位置：`/sdcard/maps/map_config.json`（每地图独立存储）
- 相关文件：`draw_Gui.cpp`

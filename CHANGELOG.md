# 更新日志

## [2026-06-27] v3.0 — 自动识别地图全面重构

### 🏗️ 架构重构

**21. 完全重写自动识别地图模块**
- **核心变更**：识别触发机制从"物体位置变化"改为"玩家传送事件检测"
- 新增 `MapDetectPhase` 状态机（LOCKED → SWITCH_DETECTED → IDENTIFYING → LOW_CONFIDENCE）
- 新增 `DetectPlayerTeleport()` 检测 XY 大幅跳跃(>80) 或 Z 大幅跳跃(>50)
- 新增 200ms 防抖机制，防止快速切图频繁触发
- 新增低置信度超时机制（3秒无匹配保持当前地图）

### 📊 评分系统增强

- 新增独立指纹数据库 `g_fingerprint_db`（从 `musicbox_stools.json` 加载）
- 评分函数 `ScoreMapFingerprints` 独立为纯函数，120分制（音乐盒40+凳子数量20+凳子位置30+钢琴30）
- 建立指纹 ID ↔ g_all_maps 索引双向映射 `g_mapidx_from_fp_id` / `g_fp_id_from_mapidx`

### 🐛 问题修复

- **🎯 修复"音乐盒误报"**：只有 LOCKED 状态下才检测物体位置变化，切换/识别中忽略所有物体变化
- **🎯 修复楼层混淆**：楼层切换通过 Z 轴跳跃独立检测，与地图切换事件解耦
- **🎯 修复快速切图抖动**：新防抖机制等待稳定后再切换

### 💡 调试增强

- 左上角常驻状态显示：`[状态] 地图名 - 状态描述 fp=ID floor=N`
- 评分调试信息改由 `g_score_debug_buf` 显示
- 新地图提示改由 `LOW_CONFIDENCE` 状态触发（连续低分2秒后弹窗）

### 🔄 兼容性

- 旧的 `map_config.json` 路径/出口数据完全兼容
- 地图纹理文件（`mapN_floor*.png`）完全兼容
- 配置文件 `overlay_config.txt` 完全兼容
- 新增指纹数据文件 `/sdcard/maps/musicbox_stools.json`

## [2026-06-27] v1.8

### 🐛 Bug 修复

**19. FindMapByPlayerPos 错误覆盖评分识别结果**
- 根因：信号间歇性丢失时，Step2 和 Step4 的 `FindMapByPlayerPos` 使用玩家坐标找到地图，
  但若某个地图的边界范围过大（覆盖全图），它会错误返回该地图索引，覆盖了评分系统正确的识
  别结果，导致地图不断跳回错误地图
- 修复：
  - Step2（信号全丢失）：不再调用 `FindMapByPlayerPos`，保持当前地图等信号恢复
  - Step4（评分不足）：只有从未识别过地图时才用 `FindMapByPlayerPos`
  - 情况C：只有 `g_last_valid_map_index < 0` 时才重置地图，否则保持当前状态

## [2026-06-27] v1.5

### 🐛 Bug 修复

**16. 评分正确但显示"地图2"的错误贴图问题**
- 根因：`g_all_maps` JSON 解析中，每层地图各占一个 slot，奇数 slot 仅 1 层（size=1）
- 但 `GetFloorFromPlayerZ(Z)` 可能返回 1，导致 `g_all_maps[idx][1]` 越界
- 越界后读到相邻 slot 的数据（恰好是"地图2"），显示名称和纹理都变成地图2
- 修复：新增 `SafeClampFloorIdx()` 函数，在所有访问 `g_all_maps` 前做楼层索引钳制
- 影响范围：`TryAutoDetectMap` 中 3 处、`LoadMapTexture`、`Draw_MapOverlay`、UI 显示
